webpack学习一

一.安装Node.js:

1.首先到node.js官网https://nodejs.org/en/下载安装node.js。
在安装完成后，我们可以进入cmd命令行模式（Windows+R）输入以下命令输入：node-v 测试是否安装成功，正确会出现版本号

2.安装好node.js之后，进行全局安装webpack：
在cmd命令模式下，运行：
npm install -g webpack

二.创建文件配置项：

1.创建项目文件夹：
1.1 在终端使用cmd命令进入该文件夹目录，运行命令：
npm init
（输入这个命令后，终端会问你一系列诸如项目名称，项目描述，作者等信息，不过不用担心，如果你不准备在npm中发布你的模块，这些问题的答案都不重要，回车默认即可）这个时候你会发现所创建的文件夹上多了一个文件名称为：package.json文件。
内容：
{
“name”: “one-webpack”,
“version”: “1.0.0”,
“description”: “”,
“main”: “index.js”,
“scripts”: {
“test”: “echo "Error: no test specified" && exit 1”
},
“author”: “”,
“license”: “ISC”,
“devDependencies”: {
“webpack”: “^4.39.3”
}
}

1.2在package.json文件已经就绪，我们在本项目中安装Webpack作为依赖包：
在终端运行命令：

npm install –save-dev webpack

执行命令之后，我们需要在该文件夹进行创建一些需要用的项目文件：
创建dist文件夹， 是项目打包之后输出的文件夹
创建src文件夹
创建/src/index.html这是项目的首页
创建/src/main.js 项目的入口文件
在所建的文件上写上内容我们就可以执行命令：
webpack .\src\main.js -o .\dist\bundle.js
这个时候我们就可以看到 我们通过webpack打包好的文件dist\bundle.js。

1.3安装 webpack-dev-server：
因为webpack打包命令每次都要输入十分麻烦，所以我们可以安装 webpack-dev-server：

cnpm i webpack-dev-server -D,同时根据提示需要同时在项目本地中安装webpack
cnpm i webpack -D。

在安装好webpack-dev-server之后，我们接下来所需要的是建一个配置webpack的文件，
每个目录下都必须有一个webpack.config.js，它的作用就好比Gulpfile.js、或者 Gruntfile.js，就是一个项目配置，告诉webpack需要做什么。
webpack.config.js：
const path = require(‘path’)
module.exports = {

entry: path.join(dirname, ‘./src/main.js’),// 入口，表示，要使用 webpack 打包哪个文件
output: { // 输出文件相关的配置
path: path.join(dirname, ‘./dist’), // 指定 打包好的文件，输出到哪个目录中去
filename: ‘bundle.js’ // 这是指定 输出的文件的名称
}
这里添加上项目的输入和输入文件,所以在我们想用webpack打包时只需要webpack命令即可打包。
