'webpack学习二'

1.webpack-dev-server 的使用

在使用 cnpm i webpack-dev-server -D 命令安装好 ‘webpack-dev-server’,
为了用webpack-dev-server,我们还需要在package.json中添加脚本:
“scripts”: {
“test”: “echo "Error: no test specified" && exit 1”,
“dev”: “webpack-dev-server –open –port 3000 –contentBase src –hot”,
}，
修改好了package.json文件之后，我们就可以使用npm run dev命令，启动项目。
webpack-dev-server会帮我们托管资源此时script标签中../dist/bundle.js路径是错误的，正确的应该是/bundle.js。

这时候我们使用html-webpack-plugin插件解决这个问题，可以在内存中根据指定的模板页面生成一份内存中的首页，同时自动把打包好的bundle注入到页面的底部，同时如果要配置插件，需要在导出的对象中挂在一个plugins节点 。

1.1 运行cnpm i html-webpack-plugin --save-dev安装到开发依赖
1.2 修改webpack.config.js配置文件如下：

const htmlWebpackPlugin = require(‘html-webpack-plugin’)
module.exports = {
plugins: [
new htmlWebpackPlugin({
template: path.join(__dirname, ‘./src/index.html’), //制定模板文件路径
filename: ‘index.html’ //指定生成页面的名称
})
],
}

2.webpack中的loader-配置处理css，scss，less。

1.在src中添加css,在/css中创建css文件
然后在main.js中添加
import ‘./css/index.css
这样导入css之类的文件会报错，因为webpack处理不了非JS文件。
这个时候我们需要安装style-loader和css-loader：
本地运行 cnpm i style-loader css-loader -D，
npm install less less-loader --save -D，
cnpm i sass-loader -D

修改webpack.config.js文件：

module:{
rules: [

        {test: /\.css$/, use:['style-loader', 'css-loader']}, //配置.css文件的第三方loader规则
        {test: /\.less$/, use:['style-loader', 'css-loader' ,'less-loader']}, //配置处理.less文件的第三方loader规则 
        {test: /\.scss/, use:['style-loader', 'css-loader', 'less-loader']}
    ]
}

webpack学习一

一.安装Node.js:

1.首先到node.js官网https://nodejs.org/en/下载安装node.js。
在安装完成后，我们可以进入cmd命令行模式（Windows+R）输入以下命令输入：node-v 测试是否安装成功，正确会出现版本号

2.安装好node.js之后，进行全局安装webpack：
在cmd命令模式下，运行：
npm install -g webpack

二.创建文件配置项：

1.创建项目文件夹：
1.1 在终端使用cmd命令进入该文件夹目录，运行命令：
npm init
（输入这个命令后，终端会问你一系列诸如项目名称，项目描述，作者等信息，不过不用担心，如果你不准备在npm中发布你的模块，这些问题的答案都不重要，回车默认即可）这个时候你会发现所创建的文件夹上多了一个文件名称为：package.json文件。
内容：
{
“name”: “one-webpack”,
“version”: “1.0.0”,
“description”: “”,
“main”: “index.js”,
“scripts”: {
“test”: “echo "Error: no test specified" && exit 1”
},
“author”: “”,
“license”: “ISC”,
“devDependencies”: {
“webpack”: “^4.39.3”
}
}

1.2在package.json文件已经就绪，我们在本项目中安装Webpack作为依赖包：
在终端运行命令：

npm install –save-dev webpack

执行命令之后，我们需要在该文件夹进行创建一些需要用的项目文件：
创建dist文件夹， 是项目打包之后输出的文件夹
创建src文件夹
创建/src/index.html这是项目的首页
创建/src/main.js 项目的入口文件
在所建的文件上写上内容我们就可以执行命令：
webpack .\src\main.js -o .\dist\bundle.js
这个时候我们就可以看到 我们通过webpack打包好的文件dist\bundle.js。

1.3安装 webpack-dev-server：
因为webpack打包命令每次都要输入十分麻烦，所以我们可以安装 webpack-dev-server：

cnpm i webpack-dev-server -D,同时根据提示需要同时在项目本地中安装webpack
cnpm i webpack -D。

在安装好webpack-dev-server之后，我们接下来所需要的是建一个配置webpack的文件，
每个目录下都必须有一个webpack.config.js，它的作用就好比Gulpfile.js、或者 Gruntfile.js，就是一个项目配置，告诉webpack需要做什么。
webpack.config.js：
const path = require(‘path’)
module.exports = {

entry: path.join(dirname, ‘./src/main.js’),// 入口，表示，要使用 webpack 打包哪个文件
output: { // 输出文件相关的配置
path: path.join(dirname, ‘./dist’), // 指定 打包好的文件，输出到哪个目录中去
filename: ‘bundle.js’ // 这是指定 输出的文件的名称
}
这里添加上项目的输入和输入文件,所以在我们想用webpack打包时只需要webpack命令即可打包。

小白使用hexo+github搭建个人博客

##1.博客环境搭建
1.1 安装Git
Windows平台：以 Win7 64位机为例
到官网下载 Git，一路默认选项安装。本文使用的是Git-2.8.1-64-bit.

1.2 安装Node.js
Windows平台：以 Win7 64位机为例
到官网（https://nodejs.org/en/）下载 Node.js，一路默认选项安装。安装完成后，我们可以进入cmd命令行模式（Windows+R）输入以下命令输入：node-v 测试是否安装成功，正确会出现版本号

1.3 安装Hexo
在我们把Git 和 Node.js 都安装好后,我们就开始安装hexo，在安装之前我们首先创建一个存放博客文件的文件夹，如 blog，然后进入 blog 文件夹，使用cmd命令行模，进到该目录，下面开始安装并使用 Hexo：

$ npm install -g hexo-cli
$ hexo init

执行命令后，我们可以在blog的文件夹上看到目录如下：

├── _config.yml
├── package.json
├── scaffolds
├── source
| ├── _drafts
| └── _posts
└── themes
目录中的_config.yml文件用于存放网站的配置信息，你可以在此配置大部分的参数；
scaffolds是存放模板的文件夹，当新建文章时，Hexo会根据scaffold来建立文件；source是资源文件夹，用于存放用户资源，themes是主题文件夹，存放博客主题，Hexo 会根据主题来生成静态页面。
在cmd命令模式执行以下命令：
$ hexo g
$ hexo s
则在浏览器中访问http:/
/localhost:4000/
然后我们安装一个扩展：

npm install hexo-deployer-git –save 为接下来部署做准备。

2.博客部署
第一步: Github注册
打开https://github.com/，在下图的框中，分别输入自己的用户名，邮箱，密码。然后前往自己刚才填写的邮箱，点开Github发送给你的注册确认信，确认注册，结束注册流程。一定要确认注册，否则无法使用gh-pages！
第二步: 创建代码库
登陆之后，点击页面右上角的加号，选择New repository：
新建代码库，入代码库创建页面：
在Repository name下填写yourname.github.io
第二步:
打开_config.yml文件，找到：

Deployment

Docs: https://hexo.io/docs/deployment.html（在文件最后）

修改内容为：deploy:
type: git
repo: https://yourname.github.io
branch: master
第三步:
hexo deploy或hexo d完成部署。
然后本地的静态网页部署到服务器上，即可通过https://GitHub用户名.github.io访问部署出来的页面。

Vue.js 安装

1、独立版本：

我们可以在 Vue.js 的官网上直接下载 vue.min.js 并用 script 标签引入。

官方网址：https://vuejs.org/js/vue.min.js

2、使用 CDN 方法：

国外比较稳定的两个 CDN，国内还没发现哪一家比较好，目前还是建议下载到本地。

1.Staticfile CDN（国内） : https://cdn.staticfile.org/vue/2.2.2/vue.min.js

2.unpkg：https://unpkg.com/vue/dist/vue.js, 会保持和 npm 发布的最新的版本一致。

3.cdnjs : https://cdnjs.cloudflare.com/ajax/libs/vue/2.1.8/vue.min.js

3、NPM 方法

3.1首先到node.js官网https://nodejs.org/en/下载安装node.js。
在安装完成后，我们可以进入cmd命令行模式（Windows+R）输入以下命令输入：node-v 测试是否安装成功，正确会出现版本号

3.2由于 npm 安装速度慢，本教程使用了淘宝的镜像及其命令 cnpm，安装使用介绍参照：使用淘宝 NPM 镜像。

npm 版本需要大于 3.0，如果低于此版本需要升级它：
进入cmd命令行模式输入以下命令输入：npm install cnpm -g。

在在用 Vue.js 构建大型应用时推荐使用 NPM 安装：
进入cmd命令行模式输入以下命令输入：cnpm install vue。
